This project uses Docker Compose and an Nginx reverse proxy to demonstrate dynamic routing with a wildcard domain: *.fbi.com.

The current page is being served using localhost via http://fbi.com

fbi.com is setup with wildcard dns that points to 127.0.0.1, we are re-using this for example purposes.

Each service runs a tiny HTTP server (using the spurin/snooze container), and can be accessed via the format: PORT.NAME.fbi.com.

For example, http://8081.rhubarb.fbi.com routes to the container named rhubarb on port 8081.
